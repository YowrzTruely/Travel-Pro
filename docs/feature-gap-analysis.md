# בקשות פיצ'רים מול מצב קיים

| בקשת פיצ'ר | מצב קיים |
|---|---|
| העלאת מסמך / מחירון | `supplierDocuments` קיים עם אחסון קבצים |
| העלאת לוגו | אין שדה לוגו בסכמה |
| משיכת נתונים מאתר ספק | לא נבנה |
| כתובת + מפה + קישורי Waze/Google | `address` + `location {lat,lng}` קיימים בסכמה, Leaflet זמין |
| אנשי קשר עם תפקידים | `supplierContacts` עם `name, role, phone, email, primary` קיים |
| קטגוריות ספקים | 8 קטגוריות קיימות, ערן צריך לשלוח מסמך להרחבה |
| נוכחות עסקית (רשתות חברתיות) | רק `websiteUrl` + `facebookUrl` — חסרים אינסטגרם, טיקטוק, לינקדאין |
| שעות פעילות (גמישות) | רק שדה טקסט חופשי `operatingHours` |
| מוצר (כותרת/פירוט/תיאור) | מוצרים קיימים אבל עם שמות שדות שונים (`name`, `description`, `notes`) |
| תמחור גמיש לפי משתתפים | תמחור לפי כמות קיים אבל רק שלב אחד (`volumeThreshold` + מחירים) |

המסמך מתאר **6 תחומי פיצ'רים** שהם שילוב של שיפורים לפיצ'רים קיימים ופונקציונליות חדשה לחלוטין.
